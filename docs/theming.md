# Theming

Единая палитра + размеры + шрифты.

Два слоя:

1. **TypeScript-константы** (`src/theme.ts`) — для логики: условные цвета, вычисляемые размеры, передача в TSX props.
2. **SCSS-переменные** (`style.scss`, шапка файла) — для стилей. По смыслу то же самое, дублируется.

Почему два — потому что SCSS компилируется отдельно, не видит TS. Если нужна одна правда — держим соответствие вручную; оба набора короткие, это не больно.

---

## Цвета (Tokyo Night)

```scss
$bg:         rgba(26, 27, 38, 0.58);  // фон плитки (polupр)
$bg-solid:   #1a1b26;                 // непрозрачный
$fg:         #c0caf5;                 // основной текст
$muted:      #7982a9;                 // второстепенный текст, подписи
$accent:     #7aa2f7;                 // hover, активные иконки
$red:        #f7768e;
$green:      #9ece6a;
$yellow:     #e0af68;
$magenta:    #bb9af7;
$cyan:       #7dcfff;
```

Как поменять палитру на другую (например, Catppuccin):

1. Открыть `style.scss` — поменять все переменные сверху.
2. Открыть `src/theme.ts` — синхронизировать те же константы.
3. Всё. Никаких `background: #123456` в коде виджетов не должно быть.

## Размеры / радиусы / паддинги

```scss
$radius-tile:   14px;
$radius-button: 10px;
```

```ts
export const SPACING = {
    xs: 4, sm: 8, md: 12, lg: 16, xl: 24,
};
```

`SPACING` — использовать как `spacing={SPACING.md}` в Stack. Хардкодить 12 — нельзя.

## Шрифты

```scss
$font-family: "JetBrainsMono Nerd Font";
$font-sm: 11px;   // подписи, мелкие метки
$font-md: 13px;   // обычный текст
$font-lg: 16px;   // заголовок плитки, имя профиля
$font-xl: 22px;   // часы
$font-hero: 36px; // огромные цифры если понадобятся
```

Nerd Font — потому что иконки. Если другой моноширинный шрифт нужен — поменять `$font-family`, но тогда glyph-range сломается, придётся искать совместимый Nerd патч.

## Classes convention

- `.tile` — базовый стиль карточки. Применяется примитивом `<Tile>`.
- `.tile.<name>-tile` — узкие переопределения под конкретный виджет (`profile-tile`, `music-tile`).
- `.<name>-<part>` — внутренние элементы (`profile-name`, `music-cover`, `task-icon`).

Правило: классы в SCSS упорядочены в том же порядке, что и виджеты в `src/widgets/` — чтобы глазами искать легко.

## Hyprland-стилизация (blur, border)

Прозрачность + blur за окном дашборда ставится не здесь, а в `hyprland.conf`:

```
layerrule = blur, ags-dashboard
layerrule = ignorealpha 0.5, ags-dashboard
```

См. `docs/hyprland.md` (раздел по интеграции).

Внутренний blur между плитками (glassmorphism) — невозможен чисто GTK. Либо blur всего слоя целиком (Hyprland), либо эффекта «матового стекла на тайле» добиться нельзя без шейдеров.
