# eww layout — уроки из реальных сессий

---

## Не адаптируй чужой шаблон — сначала спроси про контроль

**Ситуация:** взяли rxyhn-конфиг с `vexpand` на нижних тайлах. Потратили несколько итераций на попытки "выровнять" и "сделать симметрично" через костыли — min-height, дефвары, etc.

**Что нужно было сделать сразу:** услышать слова "выровняй / симметрия / хочу контролировать размеры" → остановиться → спросить "тебе нужна гибкая резина или фиксированная сетка?". Ответ "фиксированная" — убираем vexpand с самого начала и делаем нормально.

---

## vexpand — это не высота, это "возьми остаток"

`vexpand true` на тайле означает: **занять весь оставшийся вертикальный простор** после фиксированных соседей. Размером управлять нельзя — он вычисляется, а не задаётся.

Если хочешь контролировать высоту тайла через defvar → убирай `vexpand`, ставь `min-height`.

---

## GTK горизонтальный бокс = все колонки одной высоты

Три колонки дашборда живут в `(box :orientation "h" ...)`. GTK выравнивает все дочерние элементы по высоте самого высокого.

**Следствие:** нельзя независимо задать высоту нижних тайлов разных колонок через vexpand — они все будут одной высоты (высота самой высокой колонки). Единственный способ — фиксировать все тайлы через min-height.

---

## Правильная архитектура: все тайлы фиксированные

Вместо `vexpand` на нижних тайлах — дефвар для каждого:

```yuck
; _variables.yuck
(defvar profile-h     "290")
(defvar ccu-h         "130")
(defvar sticker-h     "280")
(defvar sysbar-h      "290")
(defvar clock-h       "130")
(defvar task-h        "280")
(defvar music-h       "290")
(defvar placeholder-h "410")
```

```yuck
; в defwidget — стиль вместо vexpand
(defwidget sticker []
    (box :class "tile sticker-tile"
         :style "min-height: ${sticker-h}px;"
         ...))
```

```yuck
; в col-left — без vexpand-враппера
(defwidget col-left []
    (box :orientation "v" ...
        (profile)
        (ccu-tile)
        (sticker)))   ; просто напрямую, без (box :vexpand true ...)
```

Сумма высот в каждой колонке должна примерно совпадать с высотой окна (иначе будет зазор или overflow). Крутится через eww-tweak.

---

## eww geometry не поддерживает defvar

```yuck
; НЕПРАВИЛЬНО — упадёт с "Unknown variable dash-w"
(defwindow dashboard
    :geometry (geometry :width "${dash-w}px" ...))

; ПРАВИЛЬНО — только константы
(defwindow dashboard
    :geometry (geometry :width "1200px" :height "700px" ...))
```

Geometry резолвится при парсинге конфига, до runtime defvar'ов.

---

## inline style реактивен, geometry — нет

Если нужно менять размер виджета через `eww update` → используй `:style "min-height: ${var}px;"` внутри defwidget. Это работает реактивно. Geometry окна — статична, только перезапуск.
